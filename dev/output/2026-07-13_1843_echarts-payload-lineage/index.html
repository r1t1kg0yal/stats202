What changed at each step

Jul 8 → Jul 9 · package migration plus targeted runtime ergonomics

Imports moved from ai_development.* to dashboards, core, prism_mcp, and prism_meta. The ECharts asset moved to the split checkout web path.

The engine accepted S3 manifest keys, normalized tool-output aliases, added range inputs, completed stat-grid rendering, and hardened sharing metadata and telemetry startup.

Jul 9 → Jul 11 · operations layer and context architecture rewrite

The engine gained public layout audit, typed manifest transactions, deterministic inspection, cadence synchronization, stricter popup/interaction validation, and rollback-safe mutation.

The large preflight and 1,949-line hub became a compact router and kernel. New build and diagnose owners expanded seven spokes to nine, with matching prompt suites.

Jul 11 → Jul 12 · integration truth pass

Code and guidance stopped claiming persisted-script namespace parity. Explicit imports became mandatory; only flat, non-empty CSV output counted as current-cycle dataset success.

The renderer added a complete offline HTML download. The package removed the private audit alias and aligned examples with PlotTool and the observed split-checkout asset layout.

Jul 12 → undated staging · immutable definition versioning

Engine version 0.5 records the template and both persisted scripts as one immutable definition recipe, with current/previous state and recipe hashes.

Listing and restore APIs added product-language reversion. Current-version guards joined the template SHA guard for concurrent edits.

Undated staging → canonical · reliability qualification

Canonical unified persisted-script namespaces, removed noisy runner probes, centralized clean refresh, and added transactional script edits plus helper-aware producer analysis.

It also added structured data-quality evidence, geo maps, accessible theme resolution, stronger filter/popup/layout contracts, data-grid/export behavior, and compatibility-complete package exports.
