## The Markup Pricing Formula

```
╔═══════════════════════════════════════════════════════════════╗
║  KALECKI-ANDREWS-MEANS PRICING                               ║
╠═══════════════════════════════════════════════════════════════╣
║                                                               ║
║  P = (1 + μ) · (W·L/Y)                                       ║
║    = (1 + μ) · (W/A)                                         ║
║                                                               ║
║  where:                                                       ║
║    P = Price level                                           ║
║    μ = Markup (%)                                            ║
║    W = Money wage                                            ║
║    L = Employment                                            ║
║    Y = Real output                                           ║
║    A = Labor productivity (Y/L)                              ║
║                                                               ║
║  Components:                                                  ║
║    W·L/Y = Unit Labor Cost (ULC)                            ║
║    (1+μ) = Markup factor                                     ║
║           (covers overhead + target profit)                   ║
║                                                               ║
╚═══════════════════════════════════════════════════════════════╝

    IMPLICATIONS:
    ────────────
    
    Taking logs and differentiating:
    
        π = Ẇ/W - Ȧ/A  (when μ is stable)
        
    Inflation rate = Wage growth - Productivity growth
    
    
    ┌──────────────────────────────────────────────────┐
    │                                                  │
    │  Inflation is COST-PUSH, not demand-pull        │
    │                                                  │
    │  Money accommodates wage-price dynamics         │
    │  Money does NOT cause inflation                 │
    │                                                  │
    └──────────────────────────────────────────────────┘
```

## The Horizontal AS Curve

```
    MAINSTREAM AS CURVE          POST-KEYNESIAN AS CURVE
    ═══════════════════          ═══════════════════════
    
    P                            P (or π = inflation)
    │        ╱                   │
    │      ╱  AS                 │  ────────────────────  π*
    │    ╱   (slopes up)         │  (Horizontal at current
    │  ╱                         │   inflation rate)
    │╱                           │
    └────────→ Y                 └────────→ Y
    
    Why slopes up:               Why horizontal:
    ──────────────               ──────────────
    • Diminishing returns        • Constant AVC
    • Rising MC                  • Markup pricing
    • Flexible prices            • Sticky prices (1 year)
    • Market clearing            • Price-setters, quantity-takers
    
    Implication:                 Implication:
    ────────────                 ────────────
    • ↑AD → ↑P and ↑Y           • ↑AD → ↑Y, π unchanged (short run)
    • Trade-off between          • No trade-off: can have ↑Y
      inflation & output           with stable inflation
```

## Price vs Quantity Adjustment

```
╔═══════════════════════════════════════════════════════════════╗
║  RESPONSE TO DEMAND INCREASE                                 ║
╠═══════════════════════════════════════════════════════════════╣
║                                                               ║
║  MAINSTREAM (Flex-price markets):                            ║
║    AD↑ → P↑ → Quantity follows from supply curve            ║
║                                                               ║
║  POST-KEYNESIAN (Fix-price markets):                         ║
║    AD↑ → Q↑ → Price unchanged (within pricing period)       ║
║                                                               ║
║  ─────────────────────────────────────────────────────────── ║
║                                                               ║
║  SHORT RUN (Within pricing period, e.g., 1 year):            ║
║                                                               ║
║    Demand shock  →  Output adjusts  →  Price fixed           ║
║                     (firms meet demand    (sticky)            ║
║                      from inventory or                        ║
║                      increase production)                     ║
║                                                               ║
║  NEXT PERIOD (At pricing review):                            ║
║                                                               ║
║    IF costs rose  →  Price adjusted upward                   ║
║    IF costs flat  →  Price unchanged                         ║
║                                                               ║
║  Demand affects OUTPUT                                        ║
║  Costs affect PRICES                                          ║
║                                                               ║
╚═══════════════════════════════════════════════════════════════╝
```

## Kinked Demand Curve (Coordination Failure)

```
    FIRM'S PERCEIVED DEMAND CURVE
    ═════════════════════════════
    
    P
    │
    │      If I raise P alone:
    │      demand falls sharply
    │         ╲
    │          ╲  ← Elastic
    │           ╲
    P*├───────────×  Current price
    │           ╱  ╲
    │         ╱      ╲
    │       ╱          ╲ ← Inelastic
    │     ╱  If I cut P alone:
    │                    demand rises slowly
    │
    └──────────────────────→ Q
    
    
    Implication: Don't change P unless competitors do too!
    
    
    Firm's Decision Tree:
    ────────────────────
    
    Should I raise price?
         │
         ├─ Will competitors follow?
         │   ├─ YES → Raise price (safe)
         │   └─ NO  → Don't raise (lose market share)
         │
         └─ How to know? → Wait for cost increase
                           (industry-wide signal)
                           
                           
    RESULT: Prices move with COSTS, not DEMAND
           "Coordination failure" locks in price rigidity
```

## Excess Capacity: The Buffer

```
    WHY FIRMS MAINTAIN EXCESS CAPACITY
    ══════════════════════════════════
    
    Capacity: 100 units/year
    ─────────────────────────────────────────────────────
                                              ↑
    Current production: 75 units             │ 25% excess
    ═══════════════════════════════          │ capacity
                                              ↓
    ─────────────────────────────────────────┴────────
    
    Reasons for excess capacity:
    
    1. DEMAND UNCERTAINTY
       Can meet unexpected surge without price increases
       
    2. BARRIER TO ENTRY
       "We can ramp up immediately if you enter our market"
       
    3. MAINTENANCE & ROTATION
       Some equipment always offline
       
    4. STRATEGIC FLEXIBILITY
       Can shift product mix without bottlenecks
       
       
    US DATA: Capacity utilization typically 70-85%
    ────────
    
       Never at 100% (except rare boom peaks)
       Average: ~78%
       
       ∴ Can increase output 20-30% with NO cost increase
         = FLAT AS curve over wide range
```

## Price Changes Follow Cost Changes (Not Demand)

```
    TIME SEQUENCE OF PRICING
    ════════════════════════
    
    Period t-1:
    ───────────
    Costs: W = $20/hour, A = 5 units/hour
    ULC = W/A = $4/unit
    Markup: μ = 100%
    Price: P = (1+μ)·ULC = (2.0)·($4) = $8/unit
    
    
    Period t (demand surge):
    ────────────────────────
    AD rises 20%
    
    MAINSTREAM predicts: P rises
    
    POST-KEYNESIAN: P UNCHANGED within period
                    Q rises to meet demand
                    
        Why? Costs haven't changed yet!
             Price review is annual
             Firm has excess capacity
             
        Customers: "I'll buy 20% more at $8/unit"
        Firm: "Sure!" (increase production)
        
        
    Period t+1 (costs catch up):
    ───────────────────────────
    Tight labor market → W rises to $22/hour
    Productivity unchanged: A = 5 units/hour
    
    NEW ULC = $22/5 = $4.40/unit
    
    At pricing review:
    
    NEW Price: P = (2.0)·($4.40) = $8.80/unit
    
    NOW price rises (10%), but it's COST-driven
    
    
    ┌──────────────────────────────────────────────────┐
    │                                                  │
    │  Demand affects: QUANTITY (immediately)         │
    │  Costs affect:   PRICES (with lag)              │
    │                                                  │
    └──────────────────────────────────────────────────┘
```

## The Wage-Price Spiral

```
    COST-PUSH INFLATION MECHANISM
    ═════════════════════════════
    
    [1] Wages rise (labor market tightness, union power)
             ↓
    [2] Unit labor costs rise (W/A ↑)
             ↓
    [3] Firms mark up over new ULC at pricing review
             ↓
    [4] Prices rise
             ↓
    [5] Real wages fall (W/P ↓)
             ↓
    [6] Workers demand higher nominal W to restore real wage
             ↓
    [7] Back to [1]
    
    
    RESULT: WAGE-PRICE SPIRAL
    
    Ẇ/W > Ȧ/A  →  π > 0  →  P↑  →  W/P↓  →  Ẇ/W↑  →  π↑
    
    
    Role of Money:
    ──────────────
    
    At each stage, nominal activity ↑
         ↓
    Demand for working capital ↑
         ↓
    Bank credit ↑
         ↓
    Money supply ↑
    
    M accommodates the spiral
    M does NOT cause it
    
    
    ┌──────────────────────────────────────────────────┐
    │                                                  │
    │  To stop inflation:                             │
    │                                                  │
    │  MAINSTREAM: Reduce M (tight money)             │
    │  POST-KEYNESIAN: Reduce Ẇ/W (incomes policy)   │
    │                  OR raise Ȧ/A (productivity)    │
    │                                                  │
    │  Interest rates can SLOW AD, raising unemploy-  │
    │  ment, weakening labor, reducing Ẇ/W            │
    │  But this is COSTLY (recession needed)          │
    │                                                  │
    └──────────────────────────────────────────────────┘
```
